## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

The plan MUST pass each gate before Phase 0 research and again after Phase 1 design. Cite the constitution principle by number in any waiver.

- [ ] **Principle I (Tenant Isolation):** every new table has `tenant_id UUID NOT NULL`; every new repo query is scoped by `tenant_id`; any pgvector query filters by `tenant_id`; `tenant_id` is derived from trusted context, never the request body.
- [ ] **Principle II (Layered Architecture):** routes contain no SQL; services own business logic; repositories own queries; layers are not bypassed.
- [ ] **Principle III (Bounded Agent):** no new agent tool is introduced (current set: `rag_search`, `capture_lead`, `escalate`); agent loop limits (5 iter, 4000 tok) preserved; tool calls schema-validated; tool `tenant_id` from trusted context.
- [ ] **Principle IV (Defense-in-Depth Auth):** widget tokens are signed JWTs in memory only; origin validated server-side; CORS/CSP not relied on as auth; secrets via Vault; no `.env` committed.
- [ ] **Principle V (Lean Serving & Redaction):** no `torch`/`transformers` in serving images; model artifacts validated against `model_card.json` SHA-256; logs/traces/memory store redacted text only.
- [ ] **Principle VI (Phased Build):** work fits the current phase per constitution §Phased Build; cross-phase deps go through `CONTRACT.md`.
- [ ] **Principle VII (Clean & Simple Code):** smallest change satisfying the requirement; no speculative abstractions; canonical ID names per `CONTRACT.md` §6; no `print`, async where required.

Any unchecked box requires an entry under **Complexity Tracking** with the principle cited and the simpler alternative rejected.
